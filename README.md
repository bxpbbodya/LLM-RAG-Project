# LLM RAG Question Answering

Цей проєкт є реалізацією системи Question Answering з використанням підходу
Retrieval-Augmented Generation (RAG).

Система відповідає на запитання користувача на основі зовнішніх документів,
а не лише знань мовної моделі.

Проєкт виконано в рамках курсу LLM (RAG).

---

## Опис задачі

Метою проєкту було створити вебдодаток, який:
- приймає запитання від користувача;
- знаходить релевантні фрагменти в документах;
- формує відповідь з використанням знайденого контексту.

---

## Джерело даних

Використовується власний набір текстових документів у папці `data/`.
Документи можуть бути у форматі `.txt` або `.pdf`.

---

## Chunking

Документи розбиваються на менші частини (chunks) з перекриттям.
Для цього використовується `RecursiveCharacterTextSplitter`.

Параметри:
- розмір чанка: 900 символів
- перекриття: 150 символів

---

## Retriever

Реалізовано кілька режимів пошуку релевантних фрагментів:

- BM25 — пошук за ключовими словами (rank_bm25)
- Dense retriever — семантичний пошук (Ollama embeddings + FAISS)
- Hybrid — поєднання BM25 та семантичного пошуку
- Без пошуку — LLM відповідає без контексту

Режим можна змінювати через вебінтерфейс.

---

## LLM

Використовується локальна мовна модель `mistral`, запущена через Ollama.
Зовнішні API не використовуються.

---

## Цитування

Відповіді формуються з посиланнями на використані фрагменти документів
у форматі [1], [2]. Також відображається список чанків, що були використані
для відповіді.

---

## Інтерфейс

Вебінтерфейс реалізовано за допомогою Streamlit.
Є можливість:
- ввести запитання;
- вибрати режим пошуку;
- змінити параметри генерації відповіді.
